Generic super dispatch graph patch

Goal
- Stop treating 00/23 super caller rows as projectile payload cards.
- Add a separate Super Finder that scans for the shared dispatcher/caller layout.
- Keep payload editing in projectile/super-beam rows.

Confirmed dispatcher layout
If row base points at the 00 23 bytes:
  +0x00 u16 = 0x0023 dispatch/action opcode
  +0x02 u32 = action selector / animation selector
  +0x06 u32 = variant / branch
  +0x0A u32 = phase length / duration-ish
  +0x0E u32 = param A
  +0x12 u32 = param B
  +0x16 u32 = child script link

Child target resolver:
  child_target = 0x90893000 + child_link

Examples from the current Ryu/Shinkuu dump:
  0x908D74DA selector 0x60 variant 0x0E phase 0x1E link 0x00041BE0 target 0x908D4BE0
  0x908D74F6 selector 0x61 variant 0x0E phase 0x1E link 0x00041CC0 target 0x908D4CC0
  0x908D7512 selector 0x60 variant 0x0E phase 0x1E link 0x00041C64 target 0x908D4C64
  0x908D752E selector 0x61 variant 0x0E phase 0x1E link 0x00041D44 target 0x908D4D44

- Adds fd_super_integration.py.

  Dispatch Group, Action Sel, Variant, Phase Len, Child Link, Child Target.
- Adds Rescan supers button.

Projectile scanner separation
- proj_scanner_window.py now skips 00/23 rows that match the dispatcher layout.
- Those rows are routed to the Super Finder instead.
- This prevents selector 0x60 rows from showing as fake projectile damage 96.
