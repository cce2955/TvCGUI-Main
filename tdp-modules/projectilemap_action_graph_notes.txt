Patch: projectilemap-driven action graph sniffer

Purpose:
- The 00/23 graph finder is not just for Grand Slam or supers.
- It now treats 00/23 parent -> child offset -> forward scan as the generic action/special ownership path.
- projectilemap.json is used as the expected-damage map for all specials/projectiles for the active character.

Key behavior:
- Builds a per-character damage lookup from projectilemap.json.
- For every valid 00/23 dispatch row, resolves child target.
- Scans forward from the child target.
- Promotes a damage field when the value matches projectilemap.json for that character.
- Still handles clean 35/10 child hit packets, but gates ordinary non-super hits through projectilemap to avoid random integer matches.
- If a row has no 0x160+ super name but owns a projectilemap damage value, the row is named from the projectilemap move name.

Important distinction:
- move_id_map gives action IDs/selectors and super names when available.
- projectilemap.json gives known move damage signatures for all specials/projectiles.
- The graph proves ownership; projectilemap labels the damage field.

This means the program should not hard-code Grand Slam, Mound Blast, Shinkuu, etc.
It should dynamically show:
  parent dispatch
    -> child script
       Owned Damage @address = projectilemap value
