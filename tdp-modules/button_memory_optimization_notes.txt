Patch notes: frame data workbench button row + memory optimization

UI layout:
- Split the old single long top action bar into two button rows.
- Moved Reset changed, Reset Order, and Show Bones into the always-visible top button rows.
- Removed the separate gray Reset Order / Show Bones strip so it does not consume vertical space or get visually clipped.
- Renamed the scan button to Rescan specials to match the current action-graph behavior.

Memory/performance:
- Added per-scan Dolphin memory read caching for action/special graph child windows.
- Reused larger child scan windows for child scout reads instead of reading Dolphin again.
- Avoided running the projectile scanner twice during action/special graph attachment.
- Replaced projectilemap damage matching from per-damage bytes.find loops with a single forward pass over the child window checking u16 and u32 values.
- Kept parent -> child -> forward scan behavior intact; this only changes how often memory is read and how the search is performed.
